# 開発ルール

## 作業前

- **SPEC.md を確認する**（機能仕様・タイマー管理・注意点）
- 変更対象ファイルを特定し、**関係ないファイルは読まない**
- 編集前に「どのファイルをどう変更するか」を1〜3行で説明する

## 実装方針

- **変更対象ファイルは必要最小限**に絞る
- **外部ライブラリを追加しない**（npm install 禁止）
- 大きなリファクタリングは実施前にユーザーに確認する

## 壊してはいけないもの

- **練習モードの動作**（`state.gameType !== 'battle'` の分岐）
- **バトルモードのタイマー停止処理**（`endGame` / `pauseGame` / `resetToStart` での `stopEnemyAttack()`）
- **入力欄の非表示方式**（`.input-area` は `height: 0` のまま。`display: none` にしない）

## 編集後

- 確認ポイントを **3つ以内**で出す
